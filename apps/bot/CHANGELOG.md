# Changelog

## 2026-05-22

### 1. PDF 附件解析修正
- 修正 `.pdf` 解析流程，改用目前 `pdf-parse` 套件版本對應的 API。
- 解決附件讀取時出現 `pdfParse is not a function`，避免 Bot 讀不到論文內容。

### 2. `/agent summarize_channel` 回覆格式調整
- 取消固定的「今日重點 / 待辦事項 / 未解問題」模板。
- 改為依內容自然分段，附件問題會優先用文件內容直接回答。
- 問表格或特定段落時，會優先聚焦附件中可查詢的相關片段。

### 3. `/agent` 與 `/ask` 對話延續改善
- 修正記憶路由差異，讓 `/agent` 的任務與回答可被後續 `/ask` 延續使用。
- 減少「上一句在聊論文、下一句就斷線」的情況。

### 4. `/open` 作答計數即時更新
- `/open` 發題訊息現在會顯示唯讀計數按鈕：`✅ 0`。
- 每位學生成功提交一次答案後，該題下方計數會即時 `+1`（例如 `✅ 1`、`✅ 2`）。
- 支援選擇題按鈕作答與短答/問卷提交的同步計數。

### 5. 題目可附圖片（選擇題與簡答題）
- `/add` 與 `/add_short` 新增選填 `image` 附件欄位。
- 題目圖片網址會存到 `question_bank.metadata.image_url`，題目文字仍保留在 `content`。
- `/open` 發題時若有圖片，會直接顯示在 Discord Embed 內。

### 6. 題目作答計數功能
- `/open` 題目下方新增唯讀計數按鈕（`✅ N`），可顯示目前提交人數。
- 支援選擇題按鈕作答與短答/問卷提交的同步計數。

### 7. `/open` 題目卡片精簡
- 移除開題卡片中的「分類」與「作答方式」欄位。
- 保留題目內容、作答期限、選項（或提交按鈕）與題目圖片。

## 2026-05-15

### 5. Agent 上下文用量與自動壓縮
- 新增 `/context`，老師可以查看目前頻道的上下文估計用量、預算與壓縮狀態。
- `/ask`、`@機器人`、`/agent research` 會在上下文接近上限時自動壓縮舊內容。
- 壓縮摘要會保留主題、決策、待辦、來源與限制，避免長對話直接被截斷。
- `/clear_memory` 會同步清掉上下文壓縮摘要。
- 新增 `public.chat_messages` migration，作為正式 Agent 對話記憶表。
- 新增 `/bot_status`，可查看目前 Bot instance、部署、模型與 context 設定。
- 指令紀錄新增耗時與 instance id，方便排查未回應或重複回覆。

### 1. 附件閱讀能力升級
- `@機器人` 現在可以直接讀 `.txt`、`.md`、`.pdf`、`.pptx`、`.docx`。
- `/agent` 也能搭配附件一起整理內容，不再只看聊天室文字。
- 回覆會明示哪些附件已讀、哪些略過或解析失敗。

### 2. `pptx` 簡報閱讀能力
- `pptx` 會先抽逐頁文字，再做頁面級摘要。
- Bot 會盡量整理每頁主題、主要區塊和可能的重點結論。
- 這不是完整還原 PowerPoint 視覺，但會比單純抽字更接近人工看稿。

### 3. 文件編修建議
- 新增 `/edit_doc` 指令。
- 可針對 `.md`、`.txt`、`.docx` 產生修訂清單與建議修正版。
- `@機器人` 在附檔時，也能直接處理常見的修文需求。

### 4. `/agent research` 升級成研究報告模式
- `/agent research` 不再只回幾點摘要，現在會先回 Discord 短摘要，再附上一份完整 `.md` 報告。
- 報告會固定整理成：摘要、研究背景、主要發現、比較分析、風險與限制、結論與建議、來源。
- `research` 固定使用 `gemini-3.1-pro-preview`，不跟著全域模型切換。

### 1. `@機器人` 更會接續聊天室話題
- `@機器人` 現在會先看最近的聊天室內容，再回應你。
- 預設會抓最近 20 則訊息當上下文。
- 比較適合多人正在聊天時，中途把 bot 拉進話題。

### 2. `@機器人` 的聯網跟進更自然
- 如果你先問一題，再補一句「你上網查」，bot 會盡量沿用上一句主題去查。
- 有搜尋結果時，會優先根據結果回答，不再亂回「我不能上網」。

### 3. `@機器人` 思考中提示
- bot 回覆前，會先在你的訊息上掛一個思考表情。
- 回覆完成或失敗後，表情會自動移除。

## 2026-05-07

### 1. Vertex AI 串接完成
- Bot 現在可以走 Vertex AI。
- `/ask` 和出題功能改用同一套 LLM 呼叫方式。

### 2. 模型管理功能
- 新增 `/model` 指令可切換模型（全 bot 生效）。
- 支援多個 Gemini 模型。
- `/model` 僅老師可用，且禁用 DM。
- 切換模型後自動在 `#公告` 發通知。

### 3. `@機器人` 聊天模式
- 新增 `@機器人` 對話入口。
- `@機器人` 走的是純聊天模式，不會進出題流程。
- 直接回覆 bot 訊息也能觸發，不一定要每次手動 `@`。

### 4. 圖片生成能力（兩種入口）
- 新增 `/image prompt:...`。
- `@機器人` 也可以直接要求生圖。
- 對長段圖片描述的判斷更穩。

### 5. 圖片模型與 Nano Banana 2 設定
- 圖片服務新增 `imageService`。
- 預設圖片模型改為 `nano-banana-2`。

### 6. 共享記憶升級（頻道互通）
- 記憶改成整個頻道共用，不再是每個人分開。
- 圖片上傳和 bot 生圖也會記進共享記憶。
- 草稿暫存還是各自分開，避免互相蓋掉。
- `/clear_memory` 會清除頻道共享記憶和個人草稿。

### 7. 意圖判斷全面化
- `/ask` 和 `@機器人` 都會先判斷你是要出題、投票、聊天還是生圖。
- 同時保留簡單規則做備援。

### 8. 聯網能力（SerpAPI）
- 接入 SerpAPI 到 `askAgent`。
- 只有查詢型問題才會去上網找資料。
- 回答可附來源連結。

### 9. 工作室第一版
- 新增老師限定 `/agent` 指令。
- 目前可做兩件事：整理頻道內容、上網研究主題。

### 10. `/ask` 半開放型分析升級
- `/ask` 不只是聊天，現在也能做資料分析。
- 老師可用來看班級狀況、常錯題、學生表現、排行榜和題目難度。
- 學生可用來看自己的表現、弱點和複習建議。
- 老師還是可以用 `/ask` 自然語言出題。
- 學生不能用 `/ask` 查別人資料或繞過限制出題。

### 11. Agent 分析工具與輸出模板
- 新增分析專用邏輯，讓 `/ask` 更穩定地判斷問題和整理資料。
- 分析回覆統一會給你：結論、重點、下一步建議。
- 題目分析現在可以一次抓多題資料一起整理。

### 12. 說明文與指令文案更新
- 更新 `/ask` 指令說明。
- 更新 `/help`，把老師和學生能做的事分開寫清楚。

### 13. 測試補強
- 補上自動測試。
- `npm test` 現在會先跑功能測試，再跑型別檢查。
- 目前測試有通過。
